Approche pédagogique

Pedagogical approach

Comment Dhiya structure l'apprentissage pour ce module — au-delà du transfert de contenu.

How Dhiya structures learning for this module — beyond content delivery.

Apprentissage par l'action & défense orale

Learning by doing & oral defense

Chaque séquence combine cours structuré, TP sur dépôts réels (Django, FastAPI, Cassandra) et QCM pour ancrer les concepts. Les apprenants présentent et défendent leurs choix (MVC, framework web, modèle CQL) devant le groupe — la clarté de l'argument compte autant que le code.

Each sequence blends structured teaching, labs on real repos (Django, FastAPI, Cassandra), and MCQs to lock in concepts. Learners present and defend their choices (MVC, web framework, CQL model) in front of peers — clarity of argument matters as much as code.

IA comme outil, pas comme béquille

AI as a tool, not a crutch

L'assistance IA est acceptée pour la recherche et le prototypage, à condition que l'apprenant explique, critique et étende les sorties — en particulier sur le projet d'expertise et les revues de code.

AI assistance is welcome for research and prototyping, provided learners explain, critique, and extend outputs — especially on the capstone and code reviews.

Ancrage industriel

Industry-grounded delivery

Exemples et livrables calqués sur la réalité client : APIs REST, qualité de dépôt GitHub, déploiement, maintenabilité. Le module relie la POO Python et Cassandra aux contraintes de livraison — pas seulement à des notebooks isolés.

Examples and deliverables mirror client reality: REST APIs, GitHub repo quality, deployment, maintainability. The module connects Python OOP and Cassandra to shipping constraints — not isolated notebooks alone.

Parcours adaptatif

Adaptive learner paths

Conçu pour des cohortes mixtes (ingénieurs, analystes, reconversion). Diagnostic d'entrée, parcours modulaires par séquence, études de cas et boucles de feedback — avec coordination possible avec d'autres intervenants (Kedge, SUPINFO, NEXA, Institut Saint Aspais).

Built for mixed cohorts (engineers, analysts, upskilling professionals). Entry diagnostics, modular paths per sequence, case studies, and feedback loops — with room to coordinate alongside other faculty (Kedge, SUPINFO, NEXA, Institut Saint Aspais).

  • Qualité des questions — originalité et profondeur technique sous scrutiny oral.
  • Profondeur des réponses — exactitude, honnêteté et raisonnement face aux objections.
  • Livrables techniques — QCM par séquence, lab Cassandra, projet d'expertise (API + données + dépôt GitHub documenté).
  • Quality of questions — originality and technical depth under oral scrutiny.

schema.cql — modèle colonnes
CREATE KEYSPACE iaware_course
  WITH replication = {
    'class': 'SimpleStrategy',
    'replication_factor': 3
  };

CREATE TABLE events_by_device (
  device_id text,
  event_date date,
  event_time timestamp,
  metric text,
  value double,
  PRIMARY KEY ((device_id, event_date), event_time)
);
queries.cql — batch & SELECT
BEGIN BATCH
  INSERT INTO events_by_device (...) VALUES (...);
  INSERT INTO events_by_device (...) VALUES (...);
APPLY BATCH;

SELECT event_time, metric, value
FROM events_by_device
WHERE device_id = ? AND event_date = ?
LIMIT 10;
