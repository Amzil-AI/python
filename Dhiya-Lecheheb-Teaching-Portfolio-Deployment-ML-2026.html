A practical teaching portfolio for a production-oriented machine learning course: from model artifact to web service, monitored deployment, optimization, and maintainable release workflow.

Course Positioning

A course designed for learners who already understand ML concepts and now need to make models usable, reliable, and maintainable in real environments.

Production Mindset

Students learn that deployment is not the end of the ML lifecycle. It introduces versioning, monitoring, drift, operational risk, cost, and update decisions.

Hands-On Delivery

The course is structured around applied labs: web service deployment, API design, quantization, OpenCV edge deployment, and ONNX portability.

Client Relevance

The teaching angle is practical for schools and companies: learners connect model quality with constraints such as latency, scalability, maintainability, and business risk.

Learning Outcomes

By the end of the course, learners can explain and implement the main steps required to move a model toward production readiness.

1

Deploy an ML model behind a basic web API.

2

Compare WSGI, ASGI, cloud, mobile, edge, and on-premise targets.

3

Evaluate latency, throughput, cost, size, and accuracy trade-offs.

4

Prepare a monitoring, versioning, and rollback plan.

Course Syllabus

The sequence can be delivered as a compact intensive or expanded into a longer module with project mentoring.

01

ML in Production

Lifecycle thinking, data/model/code change, continuous improvement, deployment risks, and operational expectations.

02

Model Serving and Web APIs

Exporting models, serving predictions, Flask/WSGI, FastAPI/ASGI, GET vs POST, and simple production launch commands.

03

Cloud and Deployment Targets

On-premise, IaaS, PaaS, SaaS, mobile, and edge deployment choices. Focus on constraints, ownership, security, and maintainability.

04

Optimization for Inference

Latency, throughput, cost, lightweight architectures, quantization, pruning concepts, and measurement-driven optimization.

05

Edge, Computer Vision, and ONNX

OpenCV deployment patterns, face recognition project framing, ONNX model portability, and cross-runtime inference.

06

Maintenance and Final Project

Monitoring plan, CI/CD checks, model versioning, retraining triggers, release documentation, and final project presentation.

Assessment Approach

The course evaluates production reasoning, not only model accuracy. Students must justify deployment decisions and operational safeguards.

Technical Implementation

Working service or target deployment, clean repository, reproducible setup, and a model artifact that loads and predicts reliably.

Optimization Evidence

Baseline versus optimized comparison, including latency, model size, throughput, accuracy, and target-specific trade-offs.

Maintenance Readiness

Monitoring plan, model versioning, update strategy, known risks, rollback plan, and next improvement priorities.

Teaching objective: help learners think like ML practitioners responsible for systems that must keep working after the demo.
